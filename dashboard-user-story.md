# Dashboard User Story

## Foyer CLI 简介

Foyer CLI 是一个面向个人开发者的**项目落户与活动追踪命令行工具**，核心定位是：

- **项目落户**：统一的项目初始化流程，支持 dry-run 计划、GitHub 仓库自动创建、元数据索引
- **Activity 事件溯源**：所有操作（项目创建、inbox 投递等）均记录为不可变 event，支持导出、上下文生成和跨设备拼接
- **Inbox 跨项目投递**：项目间想法、需求、反馈的异步投递机制（类似内部 Issue）
- **多目录/多设备支持**：多扫描根目录管理、跨设备项目状态同步
- **派生层构建**：从 activity events 导出全文检索索引、graphify 语料、AI 上下文等派生物

设计原则：**CLI 负责原子操作与数据写入，Dashboard 负责聚合视图与人机交互**。

## 一、Foyer CLI 现有能力 vs Dashboard 需求对照

### 1.1 Foyer CLI 现有能力全景

| 能力域            | 子命令                                 | 核心功能                                                         |
| ----------------- | -------------------------------------- | ---------------------------------------------------------------- |
| **项目初始化**    | `project init/plan`                    | 项目落户、dry-run 计划、GitHub 联动、参数化配置                  |
| **项目索引**      | `project upsert-index`, `repo list`    | 项目列表查询、元数据索引（slug/desc/lane/owner/createdAt）       |
| **Inbox 投递**    | `inbox append/send`                    | 追加全局 inbox、向指定项目投递带类型的 inbox 条目（tapped 状态） |
| **Activity 事件** | `activity append/query/context/export` | 事件追加、按项目查询、上下文生成、多目标导出                     |
| **仓库扫描**      | `repo roots/devices/status`            | 多目录扫描根管理、本地设备仓库扫描、Git 状态聚合查询             |
| **仓库就绪**      | `repo prepare/manifests`               | 项目自动 clone、跨设备 manifest 拼接                             |
| **诊断仪表盘**    | `doctor`                               | 只读 dashboard：历史数据、sidecar、派生物、本地仓库状态          |
| **全文搜索**      | `search`                               | 派生层精确引用搜索                                               |

### 1.2 Dashboard 需求定义

```
核心视角：一览所有项目 + 项目下 inbox 事项推进程度
核心属性：
  - 项目整体进度标识（0-1, 2.0, 3.0 等阶段化标记）
  - 单项目 inbox 聚合视图与推进状态
```

### 1.3 对照分析

#### ✅ Dashboard 未覆盖的 CLI 能力（现有 > 需求）

1. **跨设备同步** - devices/manifests 多设备仓库状态拼接
2. **Git 状态检查** - repo status 脏状态、分支状态查询
3. **Activity 上下文导出** - activity context 低上下文材料生成、多 target 导出
4. **全文检索** - 跨项目 activity 精确引用搜索
5. **项目自动就绪** - repo prepare 自动 clone 本地不存在的仓库
6. **多目录扫描根** - 多个项目根目录的注册与管理
7. **项目初始化流程** - 完整的项目落户 workflow（含 dry-run）

#### ❌ CLI 支撑不到的 Dashboard 能力（需求 > 现有）

| 缺失能力                  | 说明                                                                     | 当前 CLI 状态                            |
| ------------------------- | ------------------------------------------------------------------------ | ---------------------------------------- |
| **项目进度状态管理**      | 项目的 0-1 / 2.0 / 3.0 等阶段标记                                        | ProjectListItem 无进度字段，无更新机制   |
| **Inbox 完整状态机**      | inbox 项的全生命周期状态流转（tapped → triaged → doing → done/rejected） | 只有初始 tapped 状态写入，无状态变更 API |
| **跨项目 Inbox 聚合查询** | 全局 / 按项目 聚合列出所有 inbox 条目                                    | inbox 是单项目写入操作，无读取聚合 API   |
| **Inbox 分类过滤**        | 按类型（feature-request/feedback/idea/notify）、优先级、时间筛选         | 无过滤查询接口                           |
| **Activity-Inbox 关联**   | inbox 操作与 activity 事件的双向追溯                                     | 写入时单向生成 activity，无反向关联查询  |
| **项目元数据扩展**        | 截止日期、负责人、标签、优先级等富属性                                   | 只有 slug/desc/lane/owner 基础字段       |
| **Inbox 条目编辑**        | 修改标题、内容、类型、分配人                                             | 只有创建操作，无编辑/删除 API            |
| **批量操作**              | inbox 批量状态变更、批量分配                                             | 无批量操作接口                           |
| **进度统计聚合**          | 项目维度 / 全局维度的完成率计算                                          | 无统计聚合接口                           |

## 二、Dashboard 完整产品能力补全

### 2.1 核心领域模型扩展

#### 项目（Project）扩展字段

```typescript
interface ProjectDashboardMeta {
  slug: string; // 原有
  description: string; // 原有

  // 新增 - 进度与阶段
  phase: "0-1" | "1.0" | "2.0" | "3.0" | "archived"; // 项目阶段
  progress: number; // 0-100 完成百分比
  status: "planning" | "active" | "paused" | "completed" | "blocked";

  // 新增 - 时间与优先级
  startedAt?: string;
  targetDate?: string;
  priority: "low" | "medium" | "high" | "critical";

  // 新增 - 统计缓存
  inboxStats: {
    total: number;
    tapped: number;
    triaged: number;
    doing: number;
    done: number;
    rejected: number;
  };
  latestActivityAt?: string;
}
```

#### Inbox 条目（InboxItem）扩展字段

```typescript
interface InboxItem {
  id: string; // 原有
  created: string; // 原有
  title: string; // 原有
  type: "feature-request" | "feedback" | "idea" | "notify"; // 原有
  source: string; // 原有
  status: "tapped" | "triaged" | "doing" | "done" | "rejected"; // 扩展状态机

  // 新增
  assignee?: string; // 负责人
  priority: "low" | "medium" | "high" | "critical";
  labels: string[];
  dueDate?: string;
  estimatedHours?: number;
  parentTaskId?: string; // 关联父任务

  // 活动追溯
  createdAt: string;
  updatedAt?: string;
  startedAt?: string;
  completedAt?: string;

  // 关联引用
  relatedActivityIds: string[];
  comments: Comment[];
}

interface Comment {
  id: string;
  author: string;
  content: string;
  createdAt: string;
}
```

### 2.2 能力全景图

```
Dashboard 产品能力 =
  [项目管理层]
    项目一览 + 进度追踪 + 阶段管理 + 元数据编辑
  +
  [Inbox 工作流层]
    聚合列表 + 状态流转 + 分类过滤 + 批量操作 + 评论协作
  +
  [Activity 关联层]
    时间线视图 + 事件追溯 + 上下文生成
  +
  [统计分析层]
    完成率统计 + 吞吐量分析 + 瓶颈识别
  +
  [跨设备同步层]
    多设备状态合并 + 冲突解决
```

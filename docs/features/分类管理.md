# 分类管理

> Feature: 动态分类与标签管理
> Route: `/organize` (整理页) 弹层入口
> 首页: `/` (仪表盘) 仅展示
> Version: 2026-05-11

## 概述

Foyer 的分类系统从固定的 4 大类 (Goal / 工作 / 生活 / 探索) 重构为完全动态的分类模型。用户可以自由增删一级分类，为每个分类配置图标和颜色，每个分类均可添加子分类。

## 架构

### 数据模型

```
CategoryDef {
  id: string            // 唯一标识,如 "goal" | "cat_1712345678_1"
  label: string         // 显示名称
  icon: string          // Lucide 图标名,如 "Star", "Briefcase"
  color: string         // Tailwind text color class,如 "text-amber-400"
  bg: string            // Tailwind bg class,如 "bg-amber-500/10"
  subCategories: string[]  // 子分类名称列表
}

RepoTag {
  categoryId: string    // 关联的分类 ID
  subCategory?: string  // 可选的子分类名称
}
```

### localStorage 存储

| Key                       | Value                           | 说明                                        |
| ------------------------- | ------------------------------- | ------------------------------------------- |
| `foyer.categories`        | `CategoryDef[]` JSON            | 分类列表，首次读取时从旧 work-dirs 自动迁移 |
| `foyer.repo.cat.<path>`   | `RepoTag` JSON                  | 每个仓库的分类标签                          |
| `foyer.hidden-subs`       | `Record<string, string[]>` JSON | 每个分类下隐藏的子分类名集合                |
| `foyer.repo.click.<path>` | `number`                        | 仓库点击计数(未变)                          |

### 向后兼容

`readAllTags()` 自动检测旧格式 `{ category: "work", workDir: "方向A" }` 并转换为 `{ categoryId: "work", subCategory: "方向A" }`。`readCategories()` 首次调用时从 `foyer.work-dirs` 读取旧数据并构建默认分类列表。

### 交互清单

**一级分类：**

- 浏览：每张卡片显示图标 + 名称 + 仓库计数
- 添加：底部虚线框输入名称，自动轮询 8 色调色板分配颜色
- 编辑：点击铅笔 → 展开编辑区（12列图标网格 + 名称输入 + 颜色圆点）
- 删除：点击垃圾桶 → 清理该分类下所有仓库标签

**子分类：**

- 浏览：Switch toggle + 名称 + 计数，编辑/删除按钮 hover 显示
- 添加：每张分类卡片底部输入框
- 重命名：行内编辑，替换所有关联仓库的 subCategory 值
- 删除：移除子分类名，关联仓库的 subCategory 清空（保留 categoryId）
- 可见性：Switch 控制，隐藏后仓库归入父分类通用区

### 布局

- 弹层 `max-w-[70vw]`，分类卡片 `grid grid-cols-1 lg:grid-cols-2`
- 图标选择器 `grid-cols-12`，最大高度 `max-h-32` 滚动
- "添加分类"行 `lg:col-span-2` 跨满双列

## organize.tsx 关键逻辑

### leftGroups 计算 (行 469-506)

```typescript
// 按 categoryId 和 categoryId::subCategory 分组
// 隐藏的子分类不单独建窗格，仓库流入父分类
const key = tag.subCategory && !hidden?.has(tag.subCategory)
  ? `${tag.categoryId}::${tag.subCategory}`
  : tag.categoryId;

// 遍历 categories 顺序生成窗格列表
for (const cat of categories) {
  result.push({ category: cat, repos: grouped[cat.id] });          // 父分类窗格
  for (const sub of cat.subCategories) {
    if (hidden?.has(sub)) continue;
    result.push({ category: cat, subCategory: sub, repos: grouped[cat.id::sub] });
  }
}
```

## 扩展指南

### 添加新图标

编辑 `utils.ts` 的 `ICON_MAP`，从 lucide-react 导入新图标并加入映射。`ICON_OPTIONS` 自动从 keys 生成。

### 添加新颜色

编辑 `utils.ts` 的 `CATEGORY_COLORS` 数组，新增 `{ color, bg, label }` 条目。

### 分类自动布局

`leftGroups` 按 `categories` 数组顺序生成窗格。`organize.tsx` 使用 2 列 grid，首页使用 2 列 grid，超出自动换行。如需调整列数，修改对应 `grid-cols-2`。

### 旧数据迁移

首次读取分类时自动执行：`readCategories()` 检测 localStorage 是否有 `foyer.categories`，若无则从 `foyer.work-dirs` 读取旧子方向并构建默认 4 分类。旧标签格式 `{ category, workDir }` 由 `readAllTags()` 透明转换为新格式。

# Guard（守卫）

## 定位

这是主动调用的表达变形器，不是礼貌润色器。

目标只有一个：发现用户把判断、方案、决定外包给对方，或把事实问题说成审判追责的句法，并立刻改成“我判断 / 我建议 / 我决定 / 下一步”的承担式表达。

## 触发

- 用户输入 `/守卫 <文本>` 或 `/guard <文本>`。
- 用户明确要求“帮我变形”“去掉试探”“改硬一点”“拿回主动权”。
- 用户明确要求处理“阴阳感”“暗戳戳”“听起来像追责”的表达。
- 不要因为普通聊天里出现“感觉、或者、是不是”等词就自动触发；第一版只做主动调用，避免污染上下文。

## 输入处理

1. 提取待处理文本；保留原句顺序。
2. 可接受可选场景，例如 `--场景 职场评审`、`--对象 老板`，但场景只影响措辞力度，不改变诊断标准。
3. 如果没有提供文本，只问一句：“把要守卫的原文贴给我。”

## 诊断规则

逐句扫描这些信号，必须具体到词：

- 情绪降级：`感觉`、`觉得`、`好像`、`似乎`、`可能`，把判断伪装成感受。
- 漂浮提案：`或者...？`、`要不要...`、`是不是可以...`，把偏好伪装成可选项。
- 隐形决策外推：句末 `可以吗`、`行吗`、`你看呢`，前面却没有明确主张。
- 否定不给方案：`不合适`、`没必要`、`不用`，但没有接“应该怎么做”。
- 伪不确定免责声明：`不太确定`、`可能我理解不对`、`随便说说`，提前卸掉责任。
- 群体稀释：`大家看看`、`都行`、`看你们`，把 owner 融掉。
- 审判式追责：`怎么给我...`、`为什么给我...`、`给我干掉`、`谁让...`，把需求、缺陷、验收标准包进质问里。

单个词不定罪；判断的是句法结构是否把主动权交出去，或把对方推入防御。

## 输出格式

每次必须按下面四段输出，不要加寒暄：

```md
## 守卫判定

等级：红 / 黄 / 无
结论：一句话说明这段话在哪里丢了主动权。

## 告警点

| 原文 | 标记 | 逃掉的承担 | 应说出的主张 |
| ---- | ---- | ---------- | ------------ |

## 变形

直接给一版可发送文本。默认短、硬、清楚；不做社交润滑。

## 截断句

给 1-3 句以后可复用的起手式，例如“我直接下结论：”“我的判断是：”“我建议按这个方案走：”。
```

如果原文没有明显问题，仍然输出“无”，并只做轻量压缩，不强行改硬。

## 改写原则

- 情绪降级 → 明确判断。
- 漂浮提案 → 明确方案或下一步。
- 否定不给方案 → 补齐替代方案。
- 审判式追责 → 改成事实偏离、验收标准、执行指令。
- 问句只保留为明确验收问题，例如“是否按这个方案执行？”不要保留漂浮问句。
- 不替用户道歉、求饶、铺垫、圆场。
- 不输出“温和版”；只有在真实权限边界不明时，给“边界清晰版”，但仍必须有 owner。
- 硬不等于攻击；把审判、讽刺、反问删除，保留事实、标准、指令、验证要求。

## 存档

默认把每次调用沉淀为一条私有语料，方便跨会话复盘。

- 根目录：`~/.codex/guard-corpus/`
- 文件路径：`~/.codex/guard-corpus/{YYYY-MM}/{YYYYMMDD-HHMMSS}-{uuid8}.md`
- 每次调用创建一个新文件；如果路径已存在，重新生成 `uuid8`，禁止覆盖。
- 记录内容必须包含：时间、场景、原文、告警点、最终变形、截断句。
- 用户带 `--no-archive` 时，不写入存档。
